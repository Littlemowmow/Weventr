# SideQuest — Feature Mock Frames Spec

**Purpose:** Create realistic iOS mock frames for each core feature to use on the landing page, pitch deck (MH presentation, ZFellows), and App Store preview. Based on existing iOS UI designs from the Lovable spec but refined for native iOS aesthetic.

**Design Rules:**
- Use existing SideQuest design system (colors, typography, shadows from DesignTokens.swift)
- Light mode with sage/white card aesthetic (current redesign direction)
- Standard iOS patterns: SF Pro font, native navigation bars, tab bars, sheets
- 393x852px frames (iPhone 15 Pro dimensions)
- No dark mode for these mocks — light mode only to match landing page
- Real-looking data, not Lorem ipsum — use actual city names, realistic prices, real-sounding usernames

---

## Mock Frame 1: Swipe & Vote
**Screen: Discovery View — Group Vote Mode**

Layout:
- iOS native nav bar: "Barcelona Trip" title, group avatars (3-4 small circles) right-aligned
- Large swipe card (fills ~70% of screen): Photo of a real Barcelona location (e.g., Bunkers del Carmel), overlaid with:
  - "SideQuest" badge (sunset gradient, top-left)
  - Activity name: "Bunkers del Carmel" (24pt bold, white, bottom)
  - Location: "Gràcia, Barcelona" with MapPin icon
  - Info chips: "Viewpoint" · "Free" · "1-2 hrs"
- Below card: Vote indicator showing "3/4 voted" with small avatar dots (2 green checks, 1 red X, 1 gray pending)
- Bottom action bar: X (skip), Heart (like), Star (super like) — standard circular buttons
- Tab bar at bottom: Discover (active), Feed, Trip, Profile

**Key detail:** The vote indicator is the differentiator. Show that this isn't solo swiping — the group is deciding together in real-time.

## Build Instructions

For each frame, create a static image (Figma, or code-generated via SwiftUI snapshot) that:

1. Uses iPhone 15 Pro frame/bezel (can use a device mockup template)
2. Shows the iOS status bar (time, signal, battery)
3. Uses the SideQuest tab bar at bottom
4. Contains realistic data — real city names, plausible prices, actual-looking content
5. Follows iOS Human Interface Guidelines for spacing, touch targets, typography
6. Uses the current SideQuest color palette (sage backgrounds, white cards, teal accents)

**Output format:** High-res PNG at 2x (786x1704px) for each frame, plus versions placed inside device mockups for landing page use.
